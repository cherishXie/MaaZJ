## v2 变更摘要

- 将项目内文档与后续任务输出“优先使用中文”纳入 AGENTS 规则。
- 将 `assets/interface.json` 是运行与发布配置唯一优先入口写为 root `AGENTS.md` 的最高优先级规则。
- 保留 pipeline JSONC 保护，并补充：`assets/interface.json` 必须按严格 JSON 处理，因为 `install.py` 会 `json.load`。
- 明确 Python agent 状态：已由 `assets/interface.json` 配置启动，但当前业务 pipeline 对已注册 custom action/reco 的接入未证实。

- rollout 仍保持先实施 root `AGENTS.md` 与 `assets/resource/pipeline/AGENTS.md`，但 root AGENTS 提前加入 agent 简述。

## 设计目标

为 MaaZJ 后续重构与升级提供稳定、低噪声的 AI 上下文系统，让后续 agent 能快速理解项目真实运行入口、资源结构、验证方式与高风险边界。

方案分为两层：

- `AGENTS.md`：常驻上下文，负责仓库导航、安全规则、优先级规则。
- 项目 skill：按需加载的流程知识，负责复杂重构、ProjectInterface 维护、MaaFramework/MFAAvalonia 升级、资源分析工具等任务。

参考 MaaXYZ/MaaHub 的模式：根 AGENTS 保持短小，主要说明 MaaFramework 专属概念和目录入口；详细规则放到独立参考文档或 skill references 中按需加载。

## 全局语言规则

后续 AGENTS 与 skill 应加入以下规则：

- 当前项目下新增或修改的说明文档、设计文档、review 结论、任务总结，默认优先使用中文。
- 文件名、路径、API 名称、MaaFramework 字段名、命令、错误信息、代码标识保留原文。
- 当上游规范、库文档或用户明确要求英文时，可以使用英文；否则面向项目维护者的文字输出优先中文。
- 后续 agent 的最终回复和中间说明也应优先中文，除非用户另有要求。

## 当前项目事实

- MaaZJ 是 MaaFramework 资源项目，不是传统应用代码库。
- 主要业务行为位于 `assets/resource/pipeline/` 与 `assets/resource/image/`。
- `assets/interface.json` 是当前运行与发布配置的最高优先级来源。
- 根目录 `interface.json` 与 `assets/interface.json` 不一致，除非任务明确针对根目录文件，否则不要把根目录 `interface.json` 当作运行真相。
- `install.py` 会复制 `assets/interface.json` 到 `install/interface.json`，随后用 `json.load` 读取并写入版本号。这进一步证明 `assets/interface.json` 是发布入口，且它必须保持严格 JSON。
- pipeline 文件中存在 `//` 注释等 JSONC-like 内容，MaaFramework 可接受；不要用严格 JSON 工具批量格式化 pipeline。
- `assets/interface.json` 配置了启动 `{PROJECT_DIR}/agent/main.py`。
- `agent/main.py` 会启动 `AgentServer`，并通过 import 注册 `地图定位检测` custom action 与 `my_reco_222` custom recognition。
- 当前未在 pipeline 中找到对 `地图定位检测` 或 `my_reco_222` 的引用。因此 agent 不是“没用”，而是“已配置启动，但业务接入未证实”。
- `deps/` 与 `install/` 应视为 vendor/generated runtime artifacts，除非用户明确要求更新打包、运行时或发布产物。

## AGENTS 设计

### 1. 根目录 `AGENTS.md`

定位：所有 agent 进入仓库后的第一屏。

建议控制在 80-150 行，写成中文为主，路径与字段名保留原文。

#### 最高优先级规则

root `AGENTS.md` 最前面应写明：

```md
最高优先级：运行与发布配置以 `assets/interface.json` 为准。不要把根目录 `interface.json` 当作 canonical 配置，除非任务明确要求处理它。`install.py` 会复制 `assets/interface.json` 到 `install/interface.json`，并使用 `json.load` 读取，因此 `assets/interface.json` 必须保持严格 JSON。
```

#### 推荐章节

- 项目定位。
- 最高优先级运行真相。
- 文档与回复语言规则。
- 目录地图。
- 安全编辑规则。
- 验证命令。
- Python agent 当前状态。
- 重构优先级。
- 升级注意事项。

#### 必须包含的指令

- 项目文档与后续任务输出默认优先中文。
- 运行行为先看 `assets/interface.json`。
- 广泛重构或升级前阅读 `docs/agent_context/PROJECT_UNDERSTANDING_2026-06-06.md`。
- 修改资源后运行 `python .\check_resource.py .\assets\resource\`。
- `assets/interface.json` 按严格 JSON 处理。
- `assets/resource/pipeline/` 下 pipeline 可能是 JSONC-like，不要用严格 JSON 工具批量改写。
- 删除或重命名 pipeline 节点前，搜索 `assets/interface.json` 与所有 pipeline 文件中的引用。
- 不要移除重复账号/服务器 task override，除非已有生成器或受支持的配置机制替代。
- `agent/main.py` 已配置启动，但当前 custom action/reco 的业务接入未证实；修改 Python agent 前先搜索 pipeline 引用。
- 避免编辑 `deps/` 与 `install/`，除非任务明确涉及打包、运行时或发布产物。

### 2. `assets/AGENTS.md`

定位：指导 `assets/` 内 ProjectInterface、resource bundle、图片、OCR 和 agent 配置关系。

推荐章节：

- `assets/interface.json` 所有权。
- resource bundle 结构。
- ProjectInterface、pipeline、image、OCR、agent 的关系。
- JSON 与 JSONC 边界。
- 编码与格式化规则。
- 验证清单。

关键规则：

- `assets/interface.json` 是 release-facing 配置，必须是严格 JSON。
- `resource` 路径保持与 `{PROJECT_DIR}/resource` 兼容。
- 不随意重命名图片资源，因为 pipeline 可能通过文件名或逻辑名称引用它们。
- pipeline 可保留 MaaFramework 兼容的 JSONC-like 写法。
- 资源改动完成前运行 `python .\check_resource.py .\assets\resource\`。

### 3. `assets/resource/pipeline/AGENTS.md`

定位：专门指导 pipeline 编辑。

推荐章节：

- pipeline 模块边界。
- 节点命名约定。
- 常见 Maa 节点字段。
- `pipeline_override` 行为。
- JSONC 处理。
- debug 与验证流程。

建议记录的模块边界：

- 启动与 app launch。
- 服务器选择。
- 用户/账号切换。
- farm/daily flow。
- 活动 flow。
- 商店。
- 公会。
- 笔记。
- 探索。
- 起号 / early-game setup。

关键规则：

- 尽量保留已有节点名，因为 ProjectInterface tasks 与 overrides 可能引用它们。
- `FARM_` 是当前日常流水线的主要命名空间。
- 重命名或删除节点前，搜索 `assets/interface.json` 与所有 pipeline 文件。
- 谨慎处理 `next`，它会改变跨模块流转。
- 不剥离注释，不用严格 JSON 工具批量格式化 pipeline。
- 若需要工具解析 pipeline，应使用 JSONC-capable parser 或 MaaFramework 资源检查结果作为准绳。

### 4. `agent/AGENTS.md`

定位：说明 Python custom action/recognition 的当前角色。

推荐章节：

- `AgentServer` 启动路径。
- 当前注册的 custom action 与 custom recognition。
- 当前业务接入状态。
- 何时修改或新增 custom 逻辑。

关键规则：

- `assets/interface.json` 已配置启动 `agent/main.py`。
- `agent/main.py` 会注册 `地图定位检测` 与 `my_reco_222`。
- 当前未证实 pipeline 使用这些 registered names；修改前先搜索引用。
- 能用 pipeline 表达的自动化优先用 pipeline。
- 只有 pipeline 难以可靠表达时，再使用 Python custom action/recognition。
- 新增 custom 逻辑时，必须确保 pipeline 中有对应引用，并安排资源验证或手动 debug 路径。

### 延后创建的 AGENTS

以下文件暂不在 v2 首轮实施中创建：

- `.github/AGENTS.md`：用于 CI、release、artifact 命名、上游下载规则。
- `docs/AGENTS.md`：用于文档规范。
- `assets/resource/image/AGENTS.md`：用于图片采集、命名、引用检查。

## Skill 设计

review 通过后，创建项目专属 skill：

`maazj-maafw-refactor`

用途：

- MaaZJ pipeline 重构。
- `assets/interface.json` 维护或生成。
- 减少账号/服务器 task 重复。
- MaaFramework 或 MFAAvalonia 升级。
- 资源分析工具建设。
- pipeline 节点引用与图片使用审计。

### Initial Layout

初期只创建必要文件：

```text
maazj-maafw-refactor/
|-- SKILL.md
`-- references/
    |-- project-map.md
    |-- pipeline-rules.md
    |-- interface-rules.md
    `-- upgrade-checklist.md
```

说明：

- 初期不创建 `agents/openai.yaml`，除非实施时明确需要 UI metadata。
- 初期不创建 `scripts/`，避免还未稳定的流程被过早自动化。
- `SKILL.md` 只放触发说明与 references 导航，不写成长篇项目百科。

### Future Layout

流程稳定后再扩展：

```text
maazj-maafw-refactor/
|-- SKILL.md
|-- agents/
|   `-- openai.yaml
|-- references/
|   |-- project-map.md
|   |-- pipeline-rules.md
|   |-- interface-rules.md
|   `-- upgrade-checklist.md
`-- scripts/
    |-- generate-interface-tasks.py
    |-- check-pipeline-refs.py
    `-- check-image-usage.py
```

### `SKILL.md`

保持简洁，只告诉 agent 在不同任务下读哪个 reference：

- 广泛重构、升级、陌生仓库任务：读 `references/project-map.md`。
- 修改 `assets/resource/pipeline/`：读 `references/pipeline-rules.md`。
- 修改或生成 `assets/interface.json`：读 `references/interface-rules.md`。
- 变更 MaaFramework、MFAAvalonia、CI release、`deps/`、`install/`：读 `references/upgrade-checklist.md`。

### `references/project-map.md`

从现有项目理解快照中提炼：

- 运行路径：MFAAvalonia -> `interface.json` -> resource bundle -> MaaFramework pipeline -> configured Python agent。
- canonical 文件。
- 主要目录职责。
- 当前风险。
- 重构目标。
- 文档与任务输出优先中文规则。

### `references/pipeline-rules.md`

记录 pipeline 规则：

- search-before-rename。
- 节点命名空间。
- JSONC 处理。
- 模块边界。
- `next`、`expected`、recognition、action 的修改思路。
- 验证流程。

### `references/interface-rules.md`

记录 ProjectInterface 规则：

- `assets/interface.json` 是 canonical。
- 根目录 `interface.json` 当前 divergent。
- `assets/interface.json` 必须是严格 JSON。
- 账号/服务器 task 重复是已知重构目标。
- 在生成器或受支持 MFAAvalonia 选项替代前，保留 `pipeline_override` 行为。
- 只有明确运行意图时才新增或删除 tasks。

### `references/upgrade-checklist.md`

记录升级流程：

- 先冻结 baseline。
- 依赖变更前运行资源验证。
- 修改 workflow 上游引用前确认目标 MFAAvalonia upstream。
- 尽量分开升级 MaaFramework 与 MFAAvalonia。
- 升级后比较资源加载、task 展示、task 启动行为。
- 注意 CI 下载 latest artifacts 与本地 `deps/` 固定旧 artifacts 的行为差异。

### 未来脚本

至少完成一次人工重构后再添加脚本。

候选脚本：

- `generate-interface-tasks.py`：从紧凑配置生成账号/服务器 ProjectInterface tasks。
- `check-pipeline-refs.py`：检查 pipeline 文件与 `assets/interface.json` 的节点引用。
- `check-image-usage.py`：识别未被 pipeline 引用的图片资源。

## 推荐落地顺序

1. review 并批准 v2 方案。
2. 首轮只实施 root `AGENTS.md` 与 `assets/resource/pipeline/AGENTS.md`。
3. root `AGENTS.md` 中提前加入简短 agent 说明，避免后续 agent 误判 Python custom 逻辑已深度参与业务 pipeline。
4. review 首轮 AGENTS 是否足够支撑下一步重构。
5. 按需补充 `assets/AGENTS.md` 与 `agent/AGENTS.md`。
6. 创建 `maazj-maafw-refactor` skill 的 initial layout。
7. 完成人工重构验证后，再决定是否加入 future scripts。

## 开放问题与 v2 建议

- 设计目录继续使用 `docs/agent_design/`。该名称清晰，且已经承载 v1/v2 版本。
- 项目 skill 建议先设计为仓库内方案，实施时再决定放入仓库目录还是用户全局 Codex skills 目录。若目标是让本仓库自带可复用上下文，优先考虑仓库内可追踪路径；若目标是仅供当前用户调用，优先考虑全局 skills。
- 根目录 `interface.json` 建议在后续单独决策：同步、标记 stale、迁移或删除都可能影响用户习惯，不在 AGENTS 实施阶段顺手处理。
- 账号/服务器 task 生成建议先于 MaaFramework/MFAAvalonia 大升级进行。原因是它能降低 ProjectInterface 重复度，减少升级时的比较噪声。
- AGENTS 建议中文为主，保留英文 API、字段、路径和命令原文。
