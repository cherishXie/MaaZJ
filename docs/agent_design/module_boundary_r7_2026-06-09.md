Date: 2026-06-09

Source of truth: `assets/interface.json`

## 结论摘要

- 本文档消费 R2 `pipeline_flow_map_initial_2026-06-09.md`、R3 `pipeline_graph_audit_2026-06-09.md` 与 R4 `image_reference_audit_2026-06-09.md`，不修改 MaaZJ 运行资源。
- 当前 module boundary 初版应把“业务出口边”“共享 recovery 边”“共享识别资源”分开建模，避免把共享等待、弹窗关闭或图片复用误判为可直接合并的业务边。
- C 类实现任务应尽量映射到单个模块或子 flow；若需要跨越启动、选服、账号切换、特殊账号 override 或共享 recovery，必须显式升级风险与验证要求。
- 首批候选实现不应触碰登录/选服/账号切换、地图定位/野猪模板、活动开放状态、全局共享图片替换或静态不可达候选删除。
- 本任务未执行 V1-V5；本文档只提供 V0 文档交叉检查结果，不证明真实流程可达。

## 模块详表

### M1 启动/登录/选服

| 字段 | 内容 |
| --- | --- |
| 职责 | 启动 app，处理公告、服务器列表、服务器选择和开始进入家园。 |
| 代表入口 | `sub_启动杖剑传说官`、`sub_启动杖剑传说官_关闭公告`、`sub_启动杖剑传说官_切换服务器` |
| 主要出口 | `sub_启动杖剑传说官_点击开始` -> `登录进入主家园界面` -> `FARM_点击杂货店` |
| 共享节点 | `启动再等等`、`执行滑动`、`登录进入主家园界面` |
| 业务跨模块边 | 启动进入 `用户切换_点击雷霆账密登录`；选服完成进入 FARM 家园入口。 |
| 共享 recovery 边 | 多数启动节点 interrupt 到 `启动再等等`；服务器列表 interrupt 到 `执行滑动`。 |
| 禁止改动边 | 不在 C1-C5 试点中改 `sub_启动杖剑传说官_选择服务器.expected` 覆盖语义；不把启动到 FARM 的边当作普通商城边处理。 |
| 后续消费 | T6 建立 S1/S2 smoke；R8 避开该模块作为首个低风险试点。 |
